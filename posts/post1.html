Creating a Contact Form

An exercise in forms and programming.

September 01, 2016

No one exists in a bubble.

As a theme for my 5 in 5 project, I wanted to explore who I am as an MFADT student. Am I a developer? creative technolgist? Artist? One of the things that makes all of these types of work possible is communication. While I was happy with the look and feel of my blog, I felt that it was limited in the sense that there was no way for readers to engage, to give feedback or ask questions.

Process

I started out my project by writing the HTML and CSS for the blog page. Having worked as a front-end developer for the 8 months prior to starting at DT, this was the area I felt most comfortable in. I used a form structure in the HTML. My goal for the CSS was to create a simple form page, that was clear and motivated the user to participate.

Contact form HTML

Getting the Form to Work

My main resource for this form was formspree.io . This service helps you take HTML forms, collect and formatsthe JSON data before sending it to your email.

To give a higher end feel to the contact form I opted to handle some of the validation client-side using javascript, AJAX and the JQuery library.

Email Validation Using a RegEx Pattern

I use the jQuery “submit” event handler to bind to the submit button on the #contatForm portion of my HTML Page. When the submit button is pushed, the contactForm executed a function I define below. The value of each form field is stored in a variable. The result of the ajax call is also stored and later displayed in the contact form.

I then use an if/else statement to validate the form’s inputs. For example I want to check that none of the variables are blank and that the email address is valid. If the form was invalidly filled out I use JQUERY to insert some html containing an error message. If all is well, a POST is sent to formspree.io, which forwards the contents of the form it to the email I provide them.

Contact Form JS/JQUERY

Challenges, Constraints, and Criteria

Using jQuery to write the contact form was the biggest challenge for me, as it the skill I am the least familiar with. I overcame the challenge by going through several tutorials and doing my best to structure my code in a similar fashion. The constraints I gave myself for this project were one day to complete it, and that the code had to be concise so that it could be reused for future projects. The criteria were simply that the form worked and was easy to use.

While the code and design of this project are mostly complete, a huge component of this project is viewer participation. For this reason, the project will never truly be complete, as participation is ongoing.

Conclusion

Overall, I think that making my site interactive was an important first project for my 5-5. I hope to receive feedback from my peers as I continue to post my work. By finding outself within the context of our communities, in this case DT, it becames easier for one to determine who they are, and who they aspire to be. If I were to do another iteration of the project I would find a way to store subscribers and email them prompts to be answered.
